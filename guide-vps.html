Kiến Trúc Hệ Thống
Hiểu cái gì chạy ở đâu trước khi làm
📖 Đọc

Thay vì một server từ Vercel, bạn tự quản lý toàn bộ stack. Mọi thứ chạy trong Docker containers trên VPS của bạn:

🌐 Internet / Users ↓ HTTPS (port 443) ⚡ Nginx Container ← Reverse proxy, SSL termination, rate limiting ↓ HTTP (port 3000, internal) 🟣 Next.js App Container ← FlowState application ↙ ↘ 🗄️ PostgreSQL 🔴 Redis ← Database & Cache Tất cả containers trong 1 Docker network nội bộ. Chỉ Nginx mới expose port ra internet (80 và 443).
✅ Ưu điểm tự host
  • Chi phí cố định $6–20/tháng dù traffic cao
  • Toàn quyền kiểm soát server và data
  • Không bị vendor lock-in
  • SSH trực tiếp, debug dễ hơn
  • Cài thêm bất kỳ software nào
  • Học được nhiều về DevOps
⚠️ Cần lưu ý
  • Bạn tự chịu trách nhiệm bảo mật
  • Cần tự cấu hình SSL, firewall
  • Uptime phụ thuộc nhà cung cấp VPS
  • Tự setup monitoring và backup
  • Cần chút kiến thức Linux cơ bản
💡
Docker là gì? Hãy tưởng tượng mỗi service (app, database, nginx) là một "hộp" riêng biệt với mọi thứ nó cần bên trong. Docker quản lý các hộp này — khởi động, dừng, kết nối chúng với nhau. Nếu app lỗi, chỉ hộp app bị ảnh hưởng, database vẫn an toàn.

02
SSH Vào Server
Kết nối terminal từ máy tính của bạn vào VPS
⏱ 10 phút
🔑
SSH là gì? Secure Shell — cho phép bạn điều khiển server từ xa qua terminal. Như TeamViewer nhưng bằng text commands, nhanh hơn và bảo mật hơn.

Từ Mac / Linux

bash
# Mở Terminal, gõ lệnh này (thay YOUR_VPS_IP bằng IP thật) ssh root@YOUR_VPS_IP # Lần đầu sẽ hỏi: "Are you sure you want to continue connecting?" # Gõ: yes ↵ # Nếu dùng password (thay vì SSH key): ssh root@YOUR_VPS_IP # Nhập password từ email DigitalOcean gửi cho bạn

Từ Windows (dùng PowerShell hoặc PuTTY)

powershell
# Windows 10/11 đã có SSH sẵn trong PowerShell: ssh root@YOUR_VPS_IP # Hoặc dùng PuTTY (tải tại putty.org): # Host Name: YOUR_VPS_IP, Port: 22, Connection type: SSH → Open

Bạn sẽ thấy màn hình như này — nghĩa là thành công!

root@flowstate-prod:~# ← Bạn đang điều khiển server!

Tạo user không phải root (bảo mật hơn)

bash
# Tạo user mới tên "deploy" adduser deploy # → Nhập password → Enter qua các fields còn lại # Cho phép deploy dùng sudo usermod -aG sudo deploy # Copy SSH key sang user mới (nếu dùng SSH key) rsync --archive --chown=deploy:deploy ~/.ssh /home/deploy # Đăng xuất root và login bằng deploy exit ssh deploy@YOUR_VPS_IP

Cài Docker & Chuẩn Bị Server
Script tự động — 1 lệnh cài tất cả
⏱ 15 phút

Source code có sẵn script scripts/setup-vps.sh làm tất cả. Nhưng nếu chưa có code trên server, chạy thủ công:

bash
# ─── Cập nhật system ──────────────────────────────── sudo apt update && sudo apt upgrade -y # ─── Cài Docker (1 lệnh duy nhất) ────────────────── curl -fsSL https://get.docker.com | sudo sh # Cho phép user hiện tại dùng Docker không cần sudo sudo usermod -aG docker $USER newgrp docker # Apply ngay không cần logout # Kiểm tra Docker đã chạy chưa docker --version # Docker version 25.x.x docker run hello-world # Nếu in ra "Hello from Docker!" → OK! # ─── Cài Docker Compose ───────────────────────────── sudo apt install docker-compose-plugin -y docker compose version # Docker Compose version v2.x.x # ─── Cài Git ──────────────────────────────────────── sudo apt install git -y # ─── Firewall ─────────────────────────────────────── sudo ufw allow ssh sudo ufw allow http sudo ufw allow https sudo ufw --force enable sudo ufw status # Kiểm tra rules # ─── Tạo thư mục project ──────────────────────────── sudo mkdir -p /opt/flowstate sudo chown $USER:$USER /opt/flowstate
✅
Checkpoint: docker run hello-world in ra "Hello from Docker!" và ufw status hiện ACTIVE là xong.

Thêm swap (quan trọng với VPS 1GB RAM)

bash
# Swap giúp VPS không bị crash khi hết RAM khi build Docker sudo fallocate -l 2G /swapfile sudo chmod 600 /swapfile sudo mkswap /swapfile sudo swapon /swapfile # Tự động mount khi reboot echo '/swapfile none swap sw 0 0' | sudo tee -a /etc/fstab free -h # Kiểm tra — phải thấy Swap: 2.0G

Clone Code & Cấu Hình .env
Đưa source code lên server và điền thông tin bí mật
⏱ 20 phút

4.1 — Push code lên GitHub (từ máy tính của bạn)

bash — máy tính của bạn
# Trong thư mục flowstate-vps trên máy tính: git init git add . git commit -m "feat: initial FlowState VPS setup" git branch -M main # Tạo repo mới trên github.com trước, sau đó: git remote add origin https://github.com/YOUR_USERNAME/flowstate.git git push -u origin main

4.2 — Clone về VPS (trong SSH session)

bash — trong SSH
cd /opt/flowstate # Clone repo (thay bằng URL repo của bạn) git clone https://github.com/YOUR_USERNAME/flowstate.git . # Kiểm tra files đã có chưa ls -la # Phải thấy: Dockerfile, docker-compose.yml, src/, prisma/, nginx/, scripts/, ...

4.3 — Tạo file .env

bash — trong SSH
# Copy template cp .env.example .env # Mở editor (nano dễ dùng nhất) nano .env # Trong nano: # Ctrl+O → Enter để lưu # Ctrl+X để thoát

Điền đầy đủ file .env. Với các secrets, tạo bằng lệnh này:

bash
# Tạo NEXTAUTH_SECRET (chạy 1 lần, copy output vào .env) openssl rand -base64 32 # Tạo DB password mạnh openssl rand -base64 24 # Tạo Redis password openssl rand -base64 24
.env — nội dung cần điền
NEXT_PUBLIC_APP_URL="https://YOURDOMAIN.COM" NEXTAUTH_URL="https://YOURDOMAIN.COM" NEXTAUTH_SECRET="output-từ-openssl-rand" POSTGRES_DB="flowstate" POSTGRES_USER="flowstate" POSTGRES_PASSWORD="password-mạnh-ở-đây" DATABASE_URL="postgresql://flowstate:password-mạnh-ở-đây@postgres:5432/flowstate" REDIS_PASSWORD="redis-password-mạnh" REDIS_URL="redis://:redis-password-mạnh@redis:6379" GOOGLE_CLIENT_ID="xxx.apps.googleusercontent.com" GOOGLE_CLIENT_SECRET="GOCSPX-xxx" ADMIN_EMAIL="your@gmail.com"

4.4 — Cập nhật domain trong nginx config và next.config.ts

bash
# Thay YOURDOMAIN.COM bằng domain thật (1 lệnh) sed -i 's/YOURDOMAIN.COM/flowstate.app/g' nginx/conf.d/flowstate.conf sed -i 's/YOURDOMAIN.COM/flowstate.app/g' next.config.ts # Kiểm tra lại grep -n "flowstate.app" nginx/conf.d/flowstate.conf
05
Lấy SSL Certificate (HTTPS miễn phí)
Let's Encrypt — certificate tự động renew mỗi 90 ngày
⏱ 10 phút
⚠️
Bước này cần domain đã trỏ về IP VPS trước! Nếu chưa có domain, bỏ qua và làm ở bước 7. Tạm thời dùng HTTP để test.

Cách 1: Dùng script có sẵn (dễ nhất)

bash
cd /opt/flowstate bash scripts/setup-ssl.sh flowstate.app # Script sẽ tự lấy certificate cho cả flowstate.app và www.flowstate.app

Cách 2: Thủ công (để hiểu rõ hơn)

bash
# Tạo Docker volumes cho certbot docker volume create flowstate_certbot_webroot docker volume create flowstate_certbot_certs # Khởi động nginx tạm (HTTP only) để verify domain docker run -d --name nginx_temp \ -p 80:80 \ -v flowstate_certbot_webroot:/var/www/certbot \ nginx:alpine # Lấy certificate docker run --rm \ -v flowstate_certbot_certs:/etc/letsencrypt \ -v flowstate_certbot_webroot:/var/www/certbot \ certbot/certbot certonly \ --webroot --webroot-path=/var/www/certbot \ --email your@email.com \ --agree-tos --no-eff-email \ -d flowstate.app -d www.flowstate.app # Dọn dẹp nginx tạm docker stop nginx_temp && docker rm nginx_temp # Kiểm tra certificate đã tồn tại docker run --rm -v flowstate_certbot_certs:/etc/letsencrypt \ certbot/certbot certificates
✅
Thành công khi thấy: "Congratulations! Your certificate and chain have been saved at /etc/letsencrypt/live/flowstate.app/"

Trỏ Domain Về VPS
Cài DNS records để domain.com → IP VPS
⏱ 10 phút + chờ DNS

Cài DNS records tại nhà cung cấp domain

Vào trang quản lý domain (Namecheap/Inet/Cloudflare...) → DNS Management → Thêm 2 records:

DNS Records cần tạo
Type Name Value TTL A @ YOUR_VPS_IP 3600 ← domain.com → IP VPS A www YOUR_VPS_IP 3600 ← www.domain.com → IP VPS # Ví dụ: IP VPS là 167.172.100.50, domain là flowstate.app A @ 167.172.100.50 3600 A www 167.172.100.50 3600

Kiểm tra DNS đã propagate chưa

bash — máy tính của bạn
# Kiểm tra domain đã trỏ đúng chưa nslookup flowstate.app # → Phải thấy IP VPS của bạn # Hoặc dùng website: dnschecker.org # Test HTTPS sau khi DNS đã propagate curl https://flowstate.app/api/health # → {"status":"ok","timestamp":"..."} → 🎉 Thành công!
⏱
DNS mất bao lâu? Thường 5–15 phút với Cloudflare. Có thể đến 48 giờ với các nhà cung cấp khác (hiếm gặp). Dùng dnschecker.org để kiểm tra từ nhiều nơi trên thế giới.

Cập nhật Google OAuth với domain mới

Google Console
# Vào console.cloud.google.com → Credentials → OAuth 2.0 Client # Thêm vào "Authorized redirect URIs": https://flowstate.app/api/auth/callback/google
08
CI/CD Tự Động với GitHub Actions
Push code → tự động build và deploy lên VPS
⏱ 20 phút
🔄
Luồng hoạt động: Bạn sửa code trên máy → git push → GitHub Actions chạy → SSH vào VPS → git pull + docker compose build + restart → Done! Không cần làm tay nữa.

8.1 — Tạo SSH key riêng cho CI/CD

bash — máy tính của bạn
# Tạo SSH key mới chỉ dùng cho GitHub Actions ssh-keygen -t ed25519 -C "github-actions-flowstate" -f ~/.ssh/flowstate_deploy # Sẽ tạo ra 2 files: # ~/.ssh/flowstate_deploy → Private key (thêm vào GitHub Secrets) # ~/.ssh/flowstate_deploy.pub → Public key (thêm vào VPS) # Xem public key cat ~/.ssh/flowstate_deploy.pub # Xem private key (copy toàn bộ để thêm vào GitHub) cat ~/.ssh/flowstate_deploy

8.2 — Thêm public key vào VPS

bash — trong SSH session
# Mở file authorized_keys trên VPS nano ~/.ssh/authorized_keys # Paste public key từ flowstate_deploy.pub vào một dòng mới # Lưu: Ctrl+O → Enter → Ctrl+X # Set quyền đúng chmod 600 ~/.ssh/authorized_keys

8.3 — Thêm Secrets vào GitHub

  1. Vào github.com/YOUR_USERNAME/flowstate → Settings → Secrets and variables → Actions
  2. Click New repository secret, thêm lần lượt:
GitHub Secrets cần tạo
VPS_HOST = 167.172.xxx.xxx ← IP VPS VPS_USER = deploy ← Username SSH VPS_SSH_KEY = -----BEGIN OPENSSH PRIVATE KEY----- ...toàn bộ nội dung file flowstate_deploy... -----END OPENSSH PRIVATE KEY----- VPS_PORT = 22

8.4 — File CI/CD (đã có trong source code)

File .github/workflows/deploy.yml đã được tạo sẵn. Mỗi khi push lên branch main:

bash — từ máy tính khi update code
# Sửa code, sau đó: git add . git commit -m "feat: thêm tính năng mới" git push origin main # → GitHub Actions tự động: # 1. Chạy type check # 2. SSH vào VPS # 3. git pull # 4. docker compose build # 5. migrate database # 6. restart containers # 7. health check # Xem tiến trình tại: github.com/YOUR_USERNAME/flowstate/actions

Monitoring & Backup
Theo dõi server và bảo vệ dữ liệu
⏱ 15 phút

Xem tình trạng server

bash
# Tình trạng tất cả containers docker compose ps # CPU và RAM usage docker stats --no-stream # Disk usage df -h # Logs lỗi gần đây docker compose logs --tail=50 app # Nginx access logs docker exec flowstate_nginx tail -f /var/log/nginx/access.log

Setup UptimeRobot (free monitoring)

  1. Vào uptimerobot.com → Free account
  2. Add New Monitor → HTTP(s) → URL: https://flowstate.app/api/health
  3. Interval: 5 minutes → Email alert khi down
  4. Bạn sẽ nhận email ngay khi website bị down

Backup database thủ công

bash
# Backup ngay lập tức docker exec flowstate_db pg_dump -U flowstate flowstate > backup_$(date +%Y%m%d).sql gzip backup_$(date +%Y%m%d).sql # Restore từ backup gunzip backup_20241201.sql.gz docker exec -i flowstate_db psql -U flowstate flowstate < backup_20241201.sql

Backup tự động hàng ngày (script đã setup từ bước 3)

bash
# Kiểm tra cron job đã được setup chưa crontab -l # → 0 2 * * * /opt/flowstate/backup.sh >> /opt/flowstate/logs/backup.log 2>&1 # Nghĩa là: Backup lúc 2:00 AM mỗi ngày, giữ 7 ngày gần nhất # Test chạy backup ngay bash /opt/flowstate/backup.sh # Xem các file backup ls -lh /opt/flowstate/backups/

Xử Lý Sự Cố
Các lỗi thường gặp và cách fix nhanh
Tham khảo

❌ Container app không khởi động được

bash
# Xem logs chi tiết docker compose logs app # Lỗi thường gặp: "Error: DATABASE_URL is not defined" → Kiểm tra file .env "Connection refused to postgres:5432" → Chờ postgres healthy "Cannot find module '/app/.next/..." → Build lại: docker compose build app # Rebuild từ đầu (xóa cache) docker compose down docker compose build --no-cache app docker compose up -d

❌ Nginx lỗi 502 Bad Gateway

bash
# Kiểm tra app container có đang chạy không docker compose ps # Nếu app không "Up (healthy)": docker compose restart app sleep 15 curl http://localhost:3000/api/health # Test trực tiếp app # Kiểm tra nginx config docker exec flowstate_nginx nginx -t # "syntax is ok" → nginx config đúng # Nếu có lỗi → kiểm tra nginx/conf.d/flowstate.conf

❌ SSL certificate lỗi

bash
# Kiểm tra certificate còn hạn không docker run --rm -v flowstate_certbot_certs:/etc/letsencrypt \ certbot/certbot certificates # Renew thủ công docker compose run --rm certbot renew # Reload nginx sau khi renew docker exec flowstate_nginx nginx -s reload

❌ VPS hết RAM, app bị kill

bash
# Xem RAM đang dùng free -h docker stats --no-stream # Giải phóng RAM Docker không dùng docker system prune -f # Nếu vẫn không đủ → nâng cấp VPS hoặc tối ưu app # DigitalOcean: Dashboard → Resize Droplet → 2GB RAM plan ($12/month)

✅ Checklist cuối — Website đã sẵn sàng

  • Website load được tại https://flowstate.app
  • /api/health trả về {"status":"ok"}
  • Đăng nhập Google hoạt động
  • HTTPS có biểu tượng 🔒 trên trình duyệt
  • HTTP tự redirect sang HTTPS
  • docker compose ps — tất cả "Up (healthy)"
  • Push code → GitHub Actions chạy tự động
  • UptimeRobot đang monitor
  • Backup cron job đang chạy
  • Admin có thể vào /admin
